Linux介绍与安装

Linux发展史与安装

什么是Linux?

Linux是一款操作系统（OS），

现在大多数的服务器上都是使用的Linux系统，如果你想要把你开发项目发布在服务器上，或者你的工作是实施工程师，那么就必须要了解Linux系统了。

Linux系统对比Windows

| | windows | Linux |
|:—-: | |:—-: | |:—-: |
| 费用 | 收费 | 免费 |
| 软件与支持 | 质量好，但是很多收费 |开源自由软件，用户可以修改定制再发布，但是由于免费，部分软件质量和体验感偏差|
| 安全性 | 经常打补丁，系统更新，还会中木马病毒 | 相对比windows安全 |
| 操作 | 图形化界面 | 命令行操作 |
| 可定制性 | 封闭的，可定制型差 | 开源的，可定制型强 |
| 应用场景 | 个人电脑桌面 | 部署用的服务器 |

Linux发展史

1、Linux前身-Unix

  • 1968年 Multics项目

  • MIT、Bell实验室、美国通用电气有限公司走到了一起，致力于开发Multics项目。到后期由于开发进度不是很好，MIT和Bell实验室相继离开这个项目的开发，最终导致项目搁浅。

  • 1970年（Unix元年，时间戳） Unix诞生

  • 当时在开发Multics项目的时候，实验室中有一个开发成员开发了一款游戏（travel space：遨游太空），因为两个实验室相继离开项目开发，导致这名开发人员没法玩游戏，后来他提议组织人员重新在Multics项目之上重新的开发，也就出现了1970年的Unix。当时Unix操作系统是使用的汇编语言（机器语言）开发的。

  • 1973年 用C语言重写Unix

  • 因为汇编语言有一个最大的局限性：对于计算机硬件过于依赖。导致移植性不好，所以后期在1973年使用了C语言对其进行重新开发。

  • 1975年，bell实验室允许大学使用Unix操作系统用于教学作用，而不允许用于商业用途。

2、Linux诞生
人物 Linus
Linux的开发作者，Linux之父，李纳斯·托瓦兹。Linux诞生时是荷兰在校大学生。

1991年 0.0.1版本
李纳斯当时学校使用的就是Unix操作系统，然后其对系统的底层代码进行了修改，放到了学校为学生开放的网站上，原先他把文件命名写成了Linus’s Unix，后期网络管理发现之后觉得这个名字不好，自己手动的将名字改成Linux。随后其他同学下载之后发现这个版本还是挺好用的，随后都把自己代码贡献给李纳斯。

  • 1992年 0.0.2版本
  • 1994年 1.0版本
  • 2003年 2.6版本

上述所提及的版本号并不是分支版本，而是指Linux的内核版本。

3、Linux分支

分支：Linux分支有很多.
目前市面上较知名的发行版有：RedHat、CentOS、Fedora、Debian、Ubuntu、SuSE、OpenSUSE、Arch Linux、SolusOS 等。

  • ReaHat系列：性能好，稳定
  • RedHat：企业版 收费
  • CentOS：相当于RedHat的免费版
  • Ubuntu系列：有比较好的图形化界面

4、开源文化

Linux是开源的操作系统。所谓开源就是指开放源代码。

人 物 Stallman 斯特曼，开源文化的倡导人。

1983年 GNU计划（目的在于建立一套完全自由和可移植的类Unix操作系统）

1985年 FSF基金会（自由软件基金会）

1990年 Emacs、GCC（c语言的编译器）、程序库

1991年 Stallman去找Linus，商谈让Linux加入其开源计划（GNU计划）

1992年 GNU/Linux

Linux系统的安装

1、安装方式

  • 目前安装操作系统方式有2种：真机安装、虚拟机安装。
  • 真机安装：使用真实的电脑进行安装，像安装windows操作系统一样，真机安装的结果就是替换掉当前的windows操作系统；
  • 虚拟机安装：通过一些特定的手段，来进行模拟安装，并不会影响当前计算机的真实操作系统；
  • 学习或者测试使用，建议使用虚拟机安装方式。

2、虚拟机软件（了解）

什么是虚拟机？

  • 虚拟机，有些时候想模拟出一个真实的电脑环境，碍于使用真机安装代价太大，因此而诞生的一款可以模拟操作系统运行的软件。
  • 虚拟机目前有2个比较有名的产品：vmware出品的vmware workstation、oracle 出品的virtual Box。

三、Linux的目录结构
记住一句经典的话 : Linux中一切皆文件.
在windows是文件的，在Linux下同样也是文件。在windows不是文件的，在Linux下也是以文件的形式存储的；

Linux采用单一树状目录结构,最上层是根目录”/”. 注意在Linux中路径用/间隔
 
Linux没有炫目的可视化操作界面，它的操作大部分都是直接执行命令，而可执行文件都是保存在相应的目录中的，所以我们对linux的操作大多数时候都是查找和执行这些可执行文件的。
 
右键在终端中打开 输入cd / 切换到根目录，再输入ls显示目录清单


和图形化界面我们看到的是一样的，下面我们来简单认识一下这些文件夹的作用：
    1.Bin: Linux指令 例如: cp复制 date设置时间 cat浏览器文件
    2.dev: 设备管理 Linux会将硬件也映射成文件进行操作和管理 例如:cpu disc
    3.etc: 配置文件 例如: mysql的配置文件my.conf
    4.home: 一般用户目录，每新建一个一般用户就会多一个目录
    5.lib/lib64: 静态库
    6.media: 媒体(光驱,u盘)
    7.mnt: 挂载(识别其他文件系统)
    8.opt: 应用程序安装文件(环境)
    9.proc: Linux内核 请勿修改.
    10.root: root超级管理员的主目录
    11.sbin: 高权限用户才可以操作的系统管理程序
    12.selinux安全加强: 类似于360防火墙 一旦有入侵行为,就会触发此目录
    13.system: 系统 请勿修改.
    14.tmp: 缓存
    15.usr/local: 系统软件数据
    
usr不是user的缩写，其实usr是Unix Software Resource的缩写，也就是Unix操作系统软件资源所放置的目录，而不是用户的数据；所有系统默认的软件都会放置到/usr, 系统安装完时，这个目录会占用最多的硬盘容量
    16.var: 可变可扩大文件(例如: 日志)
    17.lost-found: 非法关机造成文件丢失会存在此处
    18.srv: 系统服务启动时需要的文件 请勿修改.
    19.boot: 启动Linux需要的核心文件(例如: 镜像文件,连接文件)

总结:
    \1. Linux有且只有一个根目录
    \2. Linux的文件夹都是规划好了的,不要随意摆放文件.
    Linux对于设备是以文件的形式进行管理,Linux描述一切皆是使用文件.

了解Linux系统运行级别:

  • 0: 关机
  • 1: 单用户(无密码登录root)
  • 2: 多用户无网络
  • 3: 多用户有网络
  • 4: 保留
  • 5: 图像界面
  • 6: 重启

系统运行级别配置文件位置: etc/initab
切换运行级别: init[012356]

面试题:

如果不小心丢失了记录密码的文件,自己又忘记了密码,那么如何找回?
答案:
首先进入1:单用户模式,此模式下root不需要密码就可以登录.
再运行修改密码的指令.
流程:
\1. 关机
\2. 重新启动系统
\3. 再进入系统前按enter(回车) 键
\4. 输入e进入编辑模式
\5. 选择第二个选项, 再输入一个e
\6. 输入空格1回车
\7. 此时进入单用户模式并且输入小b
\8. 输入命令修改密码完成,修改密码成功,重启.

关机命令

  • 立即关机: shutdown -h now
  • 1分钟后关机: shutdown -h 1
  • 立即重启: shutdown -r now
