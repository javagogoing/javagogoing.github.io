什么是Linux?

Linux是一款操作系统（OS），

现在大多数的服务器上都是使用的Linux系统，如果你想要把你开发项目发布在服务器上，或者你的工作是实施工程师，那么就必须要了解Linux系统了。

Linux发展史

1、Linux前身-Unix

  • 1968年 Multics项目

  • MIT、Bell实验室、美国通用电气有限公司走到了一起，致力于开发Multics项目。到后期由于开发进度不是很好，MIT和Bell实验室相继离开这个项目的开发，最终导致项目搁浅。

  • 1970年（Unix元年，时间戳） Unix诞生

  • 当时在开发Multics项目的时候，实验室中有一个开发成员开发了一款游戏（travel space：遨游太空），因为两个实验室相继离开项目开发，导致这名开发人员没法玩游戏，后来他提议组织人员重新在Multics项目之上重新的开发，也就出现了1970年的Unix。当时Unix操作系统是使用的汇编语言（机器语言）开发的。

  • 1973年 用C语言重写Unix

  • 因为汇编语言有一个最大的局限性：对于计算机硬件过于依赖。导致移植性不好，所以后期在1973年使用了C语言对其进行重新开发。

  • 1975年，bell实验室允许大学使用Unix操作系统用于教学作用，而不允许用于商业用途。

2、Linux诞生
人物 Linus
Linux的开发作者，Linux之父，李纳斯·托瓦兹。Linux诞生时是荷兰在校大学生。

李纳斯当时学校使用的就是Unix操作系统，然后其对系统的底层代码进行了修改，放到了学校为学生开放的网站上，原先他把文件命名写成了Linus’s Unix，后期网络管理发现之后觉得这个名字不好，自己手动的将名字改成Linux。随后其他同学下载之后发现这个版本还是挺好用的，随后都把自己代码贡献给李纳斯。

  • 1992年 0.0.2版本
  • 1994年 1.0版本
  • 2003年 2.6版本

上述所提及的版本号并不是分支版本，而是指Linux的内核版本。

Linux系统的安装

1、安装方式

  • 目前安装操作系统方式有2种：真机安装、虚拟机安装。
  • 真机安装：使用真实的电脑进行安装，像安装windows操作系统一样，真机安装的结果就是替换掉当前的windows操作系统；
  • 虚拟机安装：通过一些特定的手段，来进行模拟安装，并不会影响当前计算机的真实操作系统；
  • 学习或者测试使用，建议使用虚拟机安装方式。

2、虚拟机软件（了解）

什么是虚拟机？

  • 虚拟机，有些时候想模拟出一个真实的电脑环境，碍于使用真机安装代价太大，因此而诞生的一款可以模拟操作系统运行的软件。
  • 虚拟机目前有2个比较有名的产品：vmware出品的vmware workstation、oracle 出品的virtual Box。

Linux的目录结构

  • 记住一句经典的话 : Linux中一切皆文件.
  • 在windows是文件的，在Linux下同样也是文件。在windows不是文件的，在Linux下也是以文件的形式存储的；

Linux采用单一树状目录结构,最上层是根目录”/”. 注意在Linux中路径用/间隔

1.Bin: Linux指令 例如: cp复制 date设置时间 cat浏览器文件
2.dev: 设备管理 Linux会将硬件也映射成文件进行操作和管理 例如:cpu disc
3.etc: 配置文件 例如: mysql的配置文件my.conf
4.home: 一般用户目录，每新建一个一般用户就会多一个目录
5.lib/lib64: 静态库
6.media: 媒体(光驱,u盘)
7.mnt: 挂载(识别其他文件系统)
8.opt: 应用程序安装文件(环境)
9.proc: Linux内核 请勿修改.
10.root: root超级管理员的主目录
11.sbin: 高权限用户才可以操作的系统管理程序
12.selinux安全加强: 类似于360防火墙 一旦有入侵行为,就会触发此目录
13.system: 系统 请勿修改.
14.tmp: 缓存
15.usr/local: 系统软件数据
usr不是user的缩写，其实usr是Unix Software Resource的缩写，也就是Unix操作系统软件资源所放置的目录，而不是用户的数据；所有系统默认的软件都会放置到/usr, 系统安装完时，这个目录会占用最多的硬盘容量
16.var: 可变可扩大文件(例如: 日志)
17.lost-found: 非法关机造成文件丢失会存在此处
18.srv: 系统服务启动时需要的文件 请勿修改.
19.boot: 启动Linux需要的核心文件(例如: 镜像文件,连接文件)

总结:

\1. Linux有且只有一个根目录
\2. Linux的文件夹都是规划好了的,不要随意摆放文件.
Linux对于设备是以文件的形式进行管理,Linux描述一切皆是使用文件.

无界面启动

如果Linux 系统启动后直接进入图形桌面，会消耗较多的内存。一般在生产环境下的Linux 系统都不会启动图形桌面，以提高系统性能。如何设置直接进入shell界面，而不启动图形桌面,设置方法如下.

编辑/etc/inittab 文件。找到id:5:initdefault: 这一行，将5 改为3，即将它改为id:3:initdefault:。

0: 关机
1: 单用户(无密码登录root)
2: 多用户无网络
3: 多用户有网络
4: 保留
5: 图像界面
6: 重启

设置完，下次启动系统时，系统默认不会进入图形桌面。只会进入命令行的Shell 控制台。

在此界面下需要联网使用命令: ifup eth0

如果在Shell 控制台想启动图形桌面，只要执行如下命令即可:startx

权限管理

在linux中每个文件有所有者,所在组,其他组的概念.

所有者

默认是谁创建了文件,谁就是所有者,但是可以通过指令进行所有者修改.
查看文件所有者指令: ls -ahl

所在组

Linux的每个用户必须在一个组之内,不能独立于组外

文件创建者(jack)所在的组,会作为文件的所在组,如果用户在创建时没有指定组,那么默认在用户组.

创建组指令: groupadd 组名
在组下创建用户指令:
    useradd -g 组名 用户名
改变用户所在组：
    usermod -g 组名 用户名

    usermod -d 组名 用户名 改变该用 户登录的初始目录
修改文件所有者指令:
    chown 用户名 文件名
修改文件所在组指令:
    chgrp 组名 文件名
tips: 需要root权限才可以进行以上操作!

其他组

除了文件的所有者和所在组中的其他用户之外,系统的其他用户都是文件的其他组.

小结

用户对于文件来说就是3种类型：所有者 、 所在组的其他用户、 非所有者并且非同一组的用户

文件类型

r: read 读
w: write 写
x: execute 执行
chmod 751 / chmod 777

-rw-r--r-- .1
-drwxr-xr-x .2
-: 表示普通文件
d: 表示目录
l: 表示连接文件(软连接)
c: 表示字符设备(键盘,鼠标)
b: 表示块文件(硬盘)

权限说明

5.1) 第二位到第四位,表示文件所有者权限

r是Reader缩写,w是Writer的缩写,表示所有者具备读写权限.
还有一个x是execute的缩写代表执行的权限

5.2) 第五位到第七位,表示文件所在组其他用户的操作权限

表示文件所在组的用户权限
r--表示当前文件所在组的用户只有读文件的权限

5.3) 第八位到第十位,表示其他组的用户的操作权限

表示文件其他组的用户权限

5.4) “.”后面的数字

文件: 文件一般文件都是1, 表示硬链接的数
目录: 表示目录下子目录的数量

5.5) 组名后面的数字

文件: 表示文件的大小
目录: 统一4096

5.6) 时间

文件最后的修改时间

小结:

a.作用于文件时
    r--表示只有读权限
    rw-表示有读写权限
    rwx表示有读写以及执行的权限
    注意: w是表示具有写权限,但是具有写权限不代表可以删除文件,如果需要删除文件还要有文件所在目录的写权限.

b.作用于目录时
    r--表示可以通过ls命令查看目录内容
    -w-表示可以修改目录名,在目录内容进行创建和删除,如果希望修改文件,需要具备文件的写权限.
    --x表示可以进入目录
    注意: 权限也可以使用数字表示
    r = 4
    w = 2
    x = 1

可以使用多个权限加在一起的结果描述文件权限, rwx可以使用7表示.

权限管理

修改权限(chmod)

通过chmod指令可以修改文件或目录的权限

方式一: 通过+ - = 的方式

u:所有者
g:所在组的用户
o:其他组的用户
a:所有的其他人(u+g+o的总和)
字符 说明 英文全称
u 该文件的所有者权限 user
g 该文件的所在组的所有用户权限 group
o 其他用户权限 other
a 系统中所有的用户 all

示例:

u=rwx : 表示所有者可以对文件进行读写执行
g=rx : 所在组的用户可以读执行
o=x : 其他组的人只能执行
g+x: 为所在组的用户增加读权限
o-x: 为其他组的用户减少执行权限

a. 分别给所有者(读写执行权限)所在组(读执行权限)其他人(执行权限)

执行命令: chmod u=rwx,g=rx,o=x b.txt

注意: 绿色表示具有执行权限

b. 将所有者的执行权限去除,增加所在组的写权限

执行命令: chmod u-x,g+w b.txt

c. 给所有人减去执行权限

执行命令: chmod a-x b.txt

方式二:通过数字变更权限

r=4
w=2
x=1
chmod u=rwx,g=rx,o=x
chmod u=7,g=5,o=1
执行命令: chmod 751

2.修改文件所有者(chown)

允许一次性修改一个目录下所有文件的所有者,但是尽量不要修改用户目录(例如: root,/home/jack, /home/rose)

执行命令: chown -R 用户名 目录名

进程管理

概念

  • 在Linux中每个执行程序都称之为一个进程,每个进程都分配了一个ID号
  • 每一个进程都会对应一个父进程,而父进程可以复制多个子进程, init是初始进程
  • 每一个进程都有可能以两种形式存在,一种是前台,一种是后台.
  • 前台进程是用户可以在操作界面上看的到的进程.
  • 后台进程是这个进行确实正在执行,但是用户在界面上无法看到的进程.
  • 系统服务一般都属于后台进程,一般在系统开启时创建,到系统关闭才被销毁.

查看系统执行的进程

  • 指令: ps[选项]
  • ps -a: 显示当前终端的所有进程
  • ps -u: 以用户的格式显示进程信息
  • ps -x: 显示后台进程运行的参数
  • 一般常用: ps -aux来查看系统正在执行的进程
  • 由于进程较多,我们可以让查看结果进行分页显示.

ps -aux | more

在分页状态下,使用空格可以翻页,回车显示下一行,b返回上一页(此操作只对文件有效),使用q退出分页查看.

user pid %cpu %mem vsz rss
用户名 进程编号 cpu占用 内存占用 虚拟内存占用 物理内存占用

stat: 运行状态(s-sleep休眠,r-run运行,d-delay短暂等待,z-僵死进程)

服务管理

  1. 概念
    服务(service)本质就是进程,但是是运行在后台的,通常都会监听某个端口,等待其他程序的请求,比如(mysql,sshd防火墙等),因此我们又称为守护进程,是Linux中非常重要的知识点.

  2. service管理指令
    service 服务名 命令
    常用命令: start|stop|restart|status
    在centOS7.0之后使用systemctl指令优化service

  3. 控制服务
    service iptables 命令
    service mysqld strat

  4. 细节分析

    1. 关闭或者启用防火墙后立即生效,可以使用windows的telnet测试端口是否正开放.
    2. 这种方式只是临时生效,重启系统后,还是会回归到对服务的设置.
    3. 如果希望设置某个服务重启后的状态,要使用chkconfig指令.

  5. 服务自动启动

    1. 查看所有服务:
      chkconfig –list | grep xxx
    2. 根据服务名查看服务:
      chkconfig 服务名 –list
    3. 全等级设置自动启动服务:
      chkconfig 服务名 on/off
    4. 指定系统等级自动启动服务:
      chkconfig –level 5 服务名 on/off

问:

为什么需要查看运行级别再去启动对应的服务呢?

答:

因为Linux有7个运行级别,每一个服务都可以设置在7个不同的运行级别下是否需要自启动.
    0：系统停机（关机）模式: 系统默认运行级别不能设置为0，否则不能正常启动，一开机就自动关机。 
    1：单用户模式: root权限，用于系统维护，禁止远程登陆，就像Windows下的安全模式登录。
        2：多用户模式: 没有NFS网络支持。 
    3：完整的多用户文本模式: 有NFS，登陆后进入控制台命令行模式。 
    4：系统未使用: 保留一般不用，在一些特殊情况下可以用它来做一些事情。例如在笔记本电脑的电池用尽时，可以切换到这个模式来做一些设置。 
    5：图形化模式: 登陆后进入图形GUI模式或GNOME、KDE图形化界面，如X Window系统。 
    6：重启模式: 默认运行级别不能设为6，否则不能正常启动，就会一直开机重启开机重启。

查看系统所有的服务: ls -l /etc/init.d/

修改系统启动级别: vim /etc/inittab

linux中命令：区分大小写

文件/文件夹命令

cd 切换目录
cd / 进入到根目录
cd ../ 返回到上一级目录
pwd 显示当前路径
ls: 查看当前目录下的所有子目录和子文件集合

ls -l: 长文件(纵向排列结果)
ls -a: 所有文件(包含.开头的隐藏文件)
组合: ls -la

mkdir: 创建目录

mkdir -p: 一次性创建多个目录
示例: mkdir -p 目录名1 目录名2 ...

touch: 创建空文件

如：touch: a.txt
也允许一次性创建多个文件 如：touch: a.txt b.txt

rm:删除文件或目录

rm [选项] 要删除的文件或者目录
-r: 递归删除
-f: 强制删除,不提示
组合 -rf

rmdir: 移除空目录

rm -rf: 移除非空目录(递归删除)

cat: 以只读的方式打开并且查看文件内容

cat [选项] 要查看的文件
选项: -n 加入行号
cat -n 要查看的文件名|more （more表示分页显示）
注意: 使用回车换行,空格翻页

cp: 拷贝文件

cp [选项] source dest
选项: -r 递归复制整个文件夹 如：cp –r test/ newtest
选项: -f 强行覆盖
选项: -i 覆盖前先询问用户

mv：移动文件或目录/重命名

重命名: mv oldFileName newFileName
移动: mv /sourceDirectory/sourceFile /destDirectory

find: 查找文件/目录

find [搜索范围] [选项]
根据目录和名称: find /home/config.txt
根据用户: find /opt -user root
根据后缀: find /root -name *.txt

找回密码

了解Linux系统运行级别:

  • 0: 关机
  • 1: 单用户(无密码登录root)
  • 2: 多用户无网络
  • 3: 多用户有网络
  • 4: 保留
  • 5: 图像界面
  • 6: 重启

系统运行级别配置文件位置: etc/initab
切换运行级别: init[012356]

面试题:

如果不小心丢失了记录密码的文件,自己又忘记了密码,那么如何找回?
答案:
首先进入1:单用户模式,此模式下root不需要密码就可以登录.
再运行修改密码的指令.
流程:
\1. 关机
\2. 重新启动系统
\3. 再进入系统前按enter(回车) 键
\4. 输入e进入编辑模式

\5. 选择第二个选项, 再输入一个e
\6. 输入空格1回车

\7. 此时进入单用户模式并且输入小b

\8. 输入命令修改密码完成,修改密码成功,重启.

关机&重启命令

关机: shutdown
立即关机: shutdown -h now
1分钟后关机: shutdown -h 1
立即重启: shutdown -r now
    关机: halt
重启系统: reboot
同步: sync(将内存中的数据同步到磁盘)
    
tips：我们在关机之前最好先进行同步操作,防止数据丢失.

压缩与解压缩

方式一:

压缩:
gzip 文件名
-r: 递归压缩,压缩目下的所有文件

解压:
gunzip 文件名

特点: 不会保留原有文件.

方式二:
压缩:
zip 压缩后的文件名.zip 源文件名
压缩目录:
zip -r 文件名.zip /目录/
-r: 递归压缩,压缩目下的所有文件

解压: unzip 文件名
语法: unzip [选项] 文件名.zip

特点: 保留原有文件.

方式三:

打包: tar
语法: tar [选项] xxx.tar.gz
选项:
-c: 压缩产生打包文件
-v: 显示详细信息
-f: 指定打包后的文件名
-z: 打包同时压缩/解压同时解压缩
-x: 解压缩打包文件
压缩： -zcvf
解压缩：-zxvf

压缩多个文件: tar -zcvf 文件名.tar.gz 源文件1 源文件2

压缩文件夹: tar -zcvf 文件名.tar.gz 目录

压缩时不包含某些目录: tar -zcvf tomcat.tar.gz -C /home tomcat
解压到当前目录: tar -zxvf 文件名.tar.gz

解压到指定目录: tar -zxvf 文件名.tar.gz -C 目录
