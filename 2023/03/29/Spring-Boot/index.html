什么是SpringBoot？

SpringBoot提供一种快速使用Spring的方式，基于约定配置思想，可以让开发人员不必在配置于逻辑业务之间进行思维切换，从而全身心的投入到逻辑业务的代码编写中，提高开发效率。

于2014年4月发布第一个版本Spring Boot 1.0.0 是Spring最顶级项目之一

官网： https://spring.io

Spring的缺点

  1. 配置繁琐
  2. 依赖繁琐

Spring Boot功能

  1. 自动配置

    Spring Boot自动配置是一个应用启动时的过程，考虑众多因素，才决定Spring配置应该用哪个，不该用那个，过程是Spring Boot自动完成的。

  2. 起步依赖（传递依赖）

    起步依赖本质上是Maven项目对象模型，将具备某种功能的坐标都打包到一起，并提供一些默认的功能。

  3. 辅助功能

    提供了一些大型项目中常见的非功能特性，如嵌入式服务器、安全、指标、健康检测、外部配置等。

Spring Boot不是对Spring的增强，而是提供了一种快速使用Spring的方式

Spring Boot创建项目步骤：
  1. 创建Mevan项目
  2. 导入起步依赖
  3. 定义controller
  4. 编写引导类（spring Boot 项目入口）
  5. 启动测试

SpringBoot项目默认打包方式为 jar

spring-boot-starter-parent

​ 定义各种技术版本信息，组合一套最优搭配的技术版本。

​ 在各种starter中，定义了该功能需要的坐标合集，其中大部分版本信息来自于分父工程

​ 我们的工程继承parent，引入starter后，通过依赖传递，就可以简单方便的获取需要的jar，并不存在版本冲突等。

@configuration

​ 把一个java类当中配置类 == xml 配置文件（配置类）

@Value

​ 可以将声明在配置文件中的值映射到被IOC容器所管理的Baen对象里面，可以通过在Baen对象里面声明一个成员变量，在成员变量上加上注解

@EnadleAutoConfiguration

​ 开启自动配置

@ComPonentScan

​ 扫描包

starters的原理

​ 利用 Maven的特点，依赖传递、版本锁定

​ starters是依赖关系的整理和封装，是一套依赖坐标的整合，可以导入应用开发的依赖坐标更方便。

利用依赖传递的特性：帮你把依赖打包了，starer

依赖管理的原理

依赖管理（Dependency Management）

 1. 继承了spring-boot-starter-parent的好处和特点
  + 默认编译java 1.8
  + 默认编译码 UTF-8
  + 通过spring-boot-denpendencies的pom管理所有公共Starter依赖
  + spring-boot-denpendencies通过Maven的一个<dependency management>特性来实现版本管理
  + 随用随取，不用继承分类所有的starter依赖

自动配置（AutoConfiguration）原理

​ 通过启动引导类@SpringBootApplication复合注解中的@EnabelAutoConfiguration，再通过Import导入AutoConfigurationImportSelector类中selectImport方法，在方法getCandidateConfigurationurations方法中通过SpringFactoriesLoadere.loadFactoryNames扫描所有MEAT-INF包下面的spring.factories文件，这个文件包含了所有要自动配置的类，但不会全部加载，会根据Conditional..条件注解来将Bean加载到IOC容器中，配置类才会生效

  1. 配置文件的本质

    将对应框架核心的Baen，加入到IOC容器中

  2. 约定优于配置思想：

    与Spring Boot起点依赖约定

    @EanableAutoConfigurattion（约定的起点 开启自动配置）

  3. 自动配置的值怎么才能生效？

    通过@conditional.. 注解进行条件判断，通过后将 @Baen 加载到 IOC 容器中，配置类才会生效

  4. 自动配置的值在哪里？

    spring-boo-autoconfigure /MEAT-INF/spring.factories

@configurationproperties

​ 可以将配置文件中的配置自动与实体类进行映射

@Component

​ 将加上该注解的类加入到IOC容器中

三个原理分析

  • Starter：是一套依赖关系的整理和封装
    • 让我们更加专注于业务开发，无需关心依赖导入，依赖冲突，预以及依赖版本
    • 在pom文件导入starter即可使用对应的功能
  • 依赖管理：依赖管理是对依赖坐标的抽取和复用，统一管理依赖坐标的版本。
    • 实现了依赖坐标的版本管理
    • starter随用随取
    • 避免了继承所有父类starter的依赖的臃肿
    • 避免了记忆所有父类starter的麻烦
    • 避免了记忆所有starter的麻烦
  • 自动配置：预先写入配置类，封装到AutoConfiguration的jar包中，按需求加载配置信息。
    • 基于约定大于配置的设计思想
    • 极大的降低了starter应用配置的复杂度
    • 代码实现原理：
      • @SpringBootAppliation–>@EnableAutoConfiguration–>@AutoConfigurationPackage(spring-boot-autoconfigure-2.1.7.RELEASE.jar)
    • 原理的核心在于：spring-boot-autoconfigure-2.1.7.RELEASE.jar

application.yml 配置文件语法

  1. 大小写敏感
  2. 数据值前边必须有空格，作为分割符
  3. 使用缩进标识层级关系
  4. 缩进不允许使用tab，只允许空格
  5. 缩进的空格数不重要，只要相同层级的元素左对齐即可
  6. ‘#’表示注释，从这个字符一直到行尾，都会被解析器忽略
  7. 数组和集合使用’- ‘表示数组每个元素
