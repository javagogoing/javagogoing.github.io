什么是SpringBoot？

SpringBoot提供一种快速使用Spring的方式，基于约定配置思想，可以让开发人员不必在配置于逻辑业务之间进行思维切换，从而全身心的投入到逻辑业务的代码编写中，提高开发效率。

于2014年4月发布第一个版本Spring Boot 1.0.0 是Spring最顶级项目之一

官网： https://spring.io

Spring的缺点

  1. 配置繁琐
  2. 依赖繁琐

Spring Boot功能

  1. 自动配置

    Spring Boot自动配置是一个应用启动时的过程，考虑众多因素，才决定Spring配置应该用哪个，不该用那个，过程是Spring Boot自动完成的。

  2. 起步依赖（传递依赖）

    起步依赖本质上是Maven项目对象模型，将具备某种功能的坐标都打包到一起，并提供一些默认的功能。

  3. 辅助功能

    提供了一些大型项目中常见的非功能特性，如嵌入式服务器、安全、指标、健康检测、外部配置等。

Spring Boot不是对Spring的增强，而是提供了一种快速使用Spring的方式

Spring Boot创建项目步骤：
  1. 创建Mevan项目
  2. 导入起步依赖
  3. 定义controller
  4. 编写引导类（spring Boot 项目入口）
  5. 启动测试

​ 利用 Maven的特点，依赖传递、版本锁定

​ starters是依赖关系的整理和封装，是一套依赖坐标的整合，可以导入应用开发的依赖坐标更方便。

依赖管理的原理

依赖管理（Dependency Management）

 1. 继承了spring-boot-starter-parent的好处和特点
  + 默认编译java 1.8
  + 默认编译码 UTF-8
  + 通过spring-boot-denpendencies的pom管理所有公共Starter依赖
  + spring-boot-denpendencies通过Maven的一个<dependency management>特性来实现版本管理
  + 随用随取，不用继承分类所有的starter依赖

自动配置（AutoConfiguration）原理

​ 通过启动引导类@SpringBootApplication复合注解中的@EnabelAutoConfiguration开启自动配置，再通过Import导入AutoConfigurationImportSelector类中selectImport()方法，在方法getCandidateConfigurations()中通过SpringFactoriesLoader.loadFactoryNames扫描所有META-INF包下面的spring.factories文件，这个文件包含了所有要自动配置的类，但不会全部加载，会根据@Condition条件注解进行筛选，通过后将Bean加载到IOC容器中，配置类才会生效

  1. 配置文件的本质

    将对应框架核心的Baen，加入到IOC容器中

  2. 约定优于配置思想：

    与Spring Boot起点依赖约定

    @EanableAutoConfigurattion（约定的起点 开启自动配置）

  3. 自动配置的值怎么才能生效？

    通过@conditional.. 注解进行条件判断，通过后将 @Baen 加载到 IOC 容器中，配置类才会生效

  4. 自动配置的值在哪里？

    spring-boo-autoconfigure /MEAT-INF/spring.factories

@configurationproperties

@Component

​ 将加上该注解的类加入到IOC容器中

三个原理分析

  • Starter：是一套依赖关系的整理和封装
    • 让我们更加专注于业务开发，无需关心依赖导入，依赖冲突，预以及依赖版本
    • 在pom文件导入starter即可使用对应的功能
  • 依赖管理：依赖管理是对依赖坐标的抽取和复用，统一管理依赖坐标的版本。
    • 实现了依赖坐标的版本管理
    • starter随用随取
    • 避免了继承所有父类starter的依赖的臃肿
    • 避免了记忆所有父类starter的麻烦
    • 避免了记忆所有starter的麻烦
  • 自动配置：预先写入配置类，封装到AutoConfiguration的jar包中，按需求加载配置信息。
    • 基于约定大于配置的设计思想
    • 极大的降低了starter应用配置的复杂度
    • 代码实现原理：
      • @SpringBootAppliation–>@EnableAutoConfiguration–>@AutoConfigurationPackage(spring-boot-autoconfigure-2.1.7.RELEASE.jar)
    • 原理的核心在于：spring-boot-autoconfigure-2.1.7.RELEASE.jar

application.yml 配置文件语法

  1. 大小写敏感
  2. 数据值前边必须有空格，作为分割符
  3. 使用缩进标识层级关系
  4. 缩进不允许使用tab，只允许空格
  5. 缩进的空格数不重要，只要相同层级的元素左对齐即可
  6. ‘#’表示注释，从这个字符一直到行尾，都会被解析器忽略
  7. 数组和集合使用’- ‘表示数组每个元素

SpringBoot的监听器

  1. 什么是监听器？

    监听器就是监听某个对象的状态变化的组件

  2. 监听器相关概念

    监听源：被监听的对象

    监听器：监听事件源对象事件源对象的状态的变化都会触发监听器

    响应行为：监听器监听到事件源的状态变化时所涉及的功能代码

监听器类型

  • CommandLineRunner：应用程序启动完成后

  • ApplicationRunner：应用程序启动完成后

    ​ 以上两个都是通过实现接口重新run()方法后，用@Component 注解加载到IOC容器

  • ApplicationContextInitializer：应用从程序初始化前

    接口是通过resources目录下创建META-INF/spring.factories文件，将ApplicationContextInitializer的实现类交个ApplicationContextInitializer接口实现监听

  • SpringApplicationRunListtener：应用程序全阶段监听

    • starting()：开始启动
    • environmentPrepared()：环境准备完成
    • contextPrepard()：IOC容器准备完成
    • contextLoaded()：IOC容器加载完成
    • stated()：应用程序启动完成
    • running()：程序正在运行
    • failed()：程序抛出异常

    接口是通过resources目录下创建META/spring.factories文件，将SpringApplicationRunListtener接口的实现类，交给SpringApplicationRunListtener接口实现·监听（注意：该接口需要启用有参构造，参数为：SpringApplication application, String[] args ）

自动配置实现原理详解

  • RedisTemplate原理

    通过conditionalOnClass判断Redisproperties.class类是否存在，存在时会加载ResisAutoConfiguration类，在通过ConditionalOnMissingBean判断是否缺失redisTemplate，缺失会把redisTemplate()方法返回的RedisTemplate加载到IOC容器

  • redisTemplate中的host和port怎么配置的？

    从application.yml/properties文件中取所有”spring.redis”前缀取值赋值给RedisProperties对象

配置类中的条件注解@Conditional及其衍生注解

  • class类条件
    • @ConditionalOnClass==存在类条件
    • @ConditionalOnMissingClass==不存在类条件
  • 属性条件
    • @ConditionalOnProperty==属性条件，还可以为属性设置默认值
  • Bean条件
    • @ConditionalOnBean==存在Bean条件
    • @ConditionalOnMissingBean==不存在Bean条件
    • @ConditionalOnSingleCondidate==只有一个Bean条件
  • 资源条件
    • @ConditionalResource==资源条件
  • Web应用条件
    • @ConditionalOnWebApplication==web应用程序条件
    • @ConditionalOnNotWebApplication==不是web应用程序条件
  • 其他条件
    • @ConditionalOneExpression==EL表达式条件
    • @ConditionalOnJava==在特定的Java版本条件

SpringBoot启动流程

启动主程序创建一个SpringApplication对象，通过run方法进入SpringApplication的构造方法中加载配置加载资源类（resourceLoader），判断启动引导类是否为空(Assert.notNull())，判断是否为web应用(webApplicationType)，设置应用初始化器(setInitializers())，设置应用监听器(setListeners())，设置运行程序的主类(mainApplicationClass)后，又回到run方法中创建一个定时器对象并启动(StopWatch.star())用于查看项目启动过程经历的时间、创建spring IOC的容器对象(ConfigurableApplicationContex)用于返回、获取所有监听器并启动(SpringApplicationRunListenerty.starting())、设置应用参数对象(ApplicationAryguments)、准备环境变量（ConfigureableEnvironment）忽略bean信息(configureIgnorBeanInfo)、正式创建IOC容器、(通过prepareContext()方法)准备环境、refreshContext()方法刷新环境后又通过afterRefresh()将所有对象注入到IOC容器、停止定时器、执行所有监听器的SpringApplicationRunListeners.started()方法、启动完成后关闭监听器，将ConfigurableApplicationContext对象返回

生产级监控Actuator

Actuator是SpringBoot提供的用来对应用系统进行自省和监控的功能模块，借助于Actuator开发者可以很方便地对应用程序内部运行以及某些监控指标进行查看、统计、Bean加载情况、配置属性、日志信息等。

Actuator的核心是端点Endpoint，它用来监视应用程序及交互，spring-boot-actuator 中已经内置了非常多的Endpoint（health、info、beans、metrics、httptrace、shutdown等），同时也允许我们自己扩展自己的Endpoint。没Endpoint都可以启动或禁用。要远程访问Endpoint，还必须通过JMX或者HTTP进行暴露，大部分应用选择HTTP。
