安装与使用

maven是什么?

maven是apache基金会的开源项目，基于java语言开发，源于犹太语，意思知识的累计。

主要是以project object manager（POM）思想管理项目的构建、报告、文档。

maven有什么用?

构建项目

加入单元测试模块,将java文件与配置文件（.xml、 .properties、 .yml ）进行分离.
src(源文件夹)
    ├main(主程序)
        ┌java(.java文件)
        └resources(资源文件)
    ├test(测试程序)
        ┌java(.java文件)
        └resources(资源文件)
target(编译好的.class文件,以及打好的包)
pom.xml(项目依赖信息)

下载jar包

通过GAV坐标自动下载所需jar包,不需要再去官网苦苦找寻jar包下载地址.
所有常用jar包都可以从  https://mvnrepository.com/  搜索下载。

依赖管理

对于重复多次开发依赖jar包较多的工程,可以建立父工程,多个项目继承父工程即可.
可以直接依赖其他的工程，使用其他工程已经开发完毕的功能。

分模块开发

spring cloud ==> dubbo
可以将同一个工程中的不同的模块以模块工程的方式进行区分.

项目打包

可以快速将项目打包（jar、war、pom）

为什么需要使用maven?

maven提供了对项目的全方位管理,可以有效的提高开发的效率,节约磁盘的空间,最重要的是更加适合分布式项目开发.

maven的安装

下载

进入官网下载最新版本的maven: http://maven.apache.org/download.cgi#

解压

将下载好的zip压缩包解压到不包含中文的路径下.

  • bin: 主程序
  • boot: maven独立的类加载器
  • conf: settings.xml maven的全局配置
  • lib: jar包

配置

MAVEN_HOME(安装根目录)

Path(path配置bin目录)

检查

在管理员权限的cmd界面mvn -v 查看maven安装是否成功.

maven仓库

仓库就用来管理maven项目的jar包的，maven的仓库分为

  • maven中央仓库
  • 国内镜像服务器
  • 公司局域网私服
  • 本地仓库

maven配置

maven安装目录下面的settings.xml文件属于全局配置文件.(E:\apache-maven-3.x.x\conf)

修改本地仓库位置

maven默认的本地仓库地址是在C盘的用户-当前登录用户文件夹下的.m2里， 如：C:\Users\Administrator.m2
修改settings.xml文件中localRepository（本地仓库）的配置

配置镜像服务器，使用阿里云镜像服务器

修改settings.xml文件在mirrors中加入mirror（镜像服务器）内容.
    <mirror>
        <id>aliyunmaven</id>
        <mirrorOf>central</mirrorOf>
        <name>aliyun maven</name>
        <url>https://maven.aliyun.com/repository/public </url>
        </mirror>

注意

修改好全局配置文件之后，将maven安装目录下的D:\apache-maven-3.0.5\conf的settings.xml文件拷贝到新仓库根目录，仓库中的settings.xml文件和与maven安装路径下的内容最好保持一致

maven工程

IDEA配置maven

IDEA构建maven项目

注意

创建好项目后，maven会自动下载maven依赖，等待完成之后，项目结构会如下

src(源文件夹)
    ├main(主程序)
        ┌java(.java文件)
        └resources(资源文件)
    ├test(测试程序)
        ┌java(.java文件)
        └resources(资源文件)
target(编译好的.class文件,以及打好的包)
pom.xml(项目依赖信息)

如果没有java或者是resources文件文件夹， 选中main文件夹，右键，选择new–》Directory

pom.xml基本信息

想要继续添加jar包依赖

mvn中央仓库地址:http://mvnrepository.com/
可以在此网站搜索jar的坐标(组织名称,项目名称,版本号)
我们添加了依赖之后，会根据配置文件的信息，从镜像服务器中下载到本地仓库。如果下次再需要这个依赖在本地仓库中已有就可以直接复用了

maven java项目运行

打开cmd进入maven工程的pom.xml文件目录下

在cmd中输入mvn compile

如果下载时出现网络问题,会导致jar包下载失败出现.lastUpdate后缀文件,如果不清除这个文件无法重新下载.请在本地仓库搜索 *.lastUpdate 文件,手动删除之后再进行下载.

maven命令

mvn compile

编译(.java ==> .class) 生成 target目录,并下载依赖文件中指定的jar包

mvn test 

测试，执行完毕后，字节码生成目录中生成了被测试类与测试类的编译字节码和测试执行过程日志与详细报告

mvn clean

执行完毕后，字节码生成目录被删除(target目录下的.class)

mvn install

安装(打成jar包、war包,存放在在maven本地仓库中根据项目本身的坐标新建文件夹保存)经历（编译，测试，打包，将打好的包装入本地仓库中）

mvn clean compile

组合指令，先执行clean，再执行compile，通常应用于上线前执行，清除测试类

mvn clean test

组合指令，先执行clean，再执行test，通常应用于测试环节

mvn clean package

组合指令，先执行clean，再执行package，将项目打包，通常应用于发布前
执行过程
    清理————清空环境
    编译————编译源码
    测试————测试源码
    打包————将编译的非测试类打包

mvn clean install

查看仓库，当前项目被发布到仓库中
组合指令，先执行clean，再执行install，将项目打包，通常应用于发布前
执行过程
    清理————清空环境
    编译————编译源码
    测试————测试源码
    打包————将编译的非测试类打包
    部署————将打好的包发布到资源仓库中

属性设置

properties用于定义一些常量,属性表述类似于EL表达式.

<properties>
    <commons.io>2.2</commons.io>
    </properties>
    <dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>commons-io</groupId>
            <artifactId>commons-io</artifactId>
            <version>${commons.io}</version>
        </dependency>
    </dependencies>
    </dependencyManagement>
