基本概念

  • 程序(program)

    为完成特定任务，用某种语言编写的一组指令集合。即指一段静态的代码，静态对象

  • 进程(process)

    程序一次的执行过程，或正在运行的一个程序，是一个动态的过程（它有自身的生产、存在和消亡的过程 – 生命周期）

  • 线程(thread)

    进程进一步细化就是线程，一个程序内部的一条执行路径，一个进程同时执行多个线程，就是多线程

    线程就是调度和执行的单位，每个线程都有独立的运行栈和程序计算器（pc）线程开销小，一个进程的多个内存共享相同的内存单元/内存地址空间–>它们从一堆中分配对象，可以访问相同的对象和变量，这就使得线程中通信简便、高效。但多个线程操作共享的系统资源可能会带来安全隐患。

多线程实现

  • Thread

    通过继承Thread抽象类run方法实现多线程(extends)

  • Runnable

    通过实现Runnable接口重写run()方法实先(implements)

线程发生安全隐患解决方案

  • 同步代码块

    synchronized(同步监听器){// 需要被同步的代码}

    同步监听：俗称锁任何一个类的对象，都可以来充当锁，多线程必须共同一把锁

  • 同步方法

    使用synchronized修饰方法就叫作同步方法，保证A线程执行该方法的时候，其他线程只能在方法外面等着。非static方法，同步锁就是this/是static方法，当前方法所在类的字节码对象就是同步锁

  • 使用Lock锁

    java.util.concurrent.lock.Lock 接口是控制多线程对资源进行访问的工具。锁对共享资源的独占访问，每次只有一个线程的Lock对象加锁，线程开始访问共享资源应先获得Lock对象。

    ReentrantLock类实现了Lock，它拥有与synchronized相同的并发性和内存语义，在实现线程安全控制中，比较常用的是RenntranLock，可以显示加锁，释放锁

  • synchronized与Lock的异同

    相同

    ​ 二者都可以解决线程安全问题

    不同

    ​ synchronized机制在执行完相应的同步代码以后，自动释放同步监视器

    ​ Lock需要手动启动同步（lock），同时结束同步代码以后也需要手动的实现（unlock()）

  • 单例模式中的懒汉模式线程安全

    使用synchronized，锁定类确保唯一性

  • 使用顺序

    Lock –> 同步代码块（已尽进入方法体，分配了相对应的资源）–>同步方法（在方法体之外）

死锁

​ 不同线程占用对方需要的同步资源不放弃，都在等待对方放弃自己想要的同步资源，就形成了线程的死锁

​ 出现线程死锁不会出现异常，不会出现提示，只是所有的线程都处于阻塞状态，无法继续

线程的通信

  • wait()

    执行此方法，当前线程就会进入堵塞状态，并释放同步监视器

  • notify()

    执行此方法，就会唤醒被wait的的一个线程，如果有多个线程就会先唤醒优先级高的那个

  • notifyAll()

    执行此方法就会唤醒所有被wait的线程

三个方法都是java.lang.Object类中的

必须使用在代码块同步和方法同步中

调用者必须是同步代码或方法同步中的监听器

sleep()和wait的异同？

相同点：

​ 一旦执行方法，都会使得当前线程进入一个阻塞状态

不同点：

​ 1）两个方法声明的位置不同：Thread类中声明sleep()，Object类中声明wait().

​ 2）调用的要求不同：sleep()可以在任何想要的场景下调用，wait()必须使用在同步代码块和同步方法中。

​ 3）关于是否释放同步监听器：如果两个都使用在同步代码块或同步方法中sleep()不会释放锁，wait()会释放锁。
